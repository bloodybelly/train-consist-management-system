# 🚆 Train Consist Management System

## 📌 Overview

This project simulates a real-world railway train consist management system using Core Java and Data Structures.

It demonstrates how dynamic data structures can efficiently manage train coach operations such as addition, removal, validation, and recovery.

---

## 🎯 Objectives

* Apply data structures to real-world problems
* Design a modular and scalable Java system
* Simulate railway coach management operations

---

## 🧠 Data Structures Used

### 🔹 Doubly Linked List

* Used to manage train coaches
* Allows efficient insertion and deletion
* Supports forward and backward traversal

### 🔹 Stack

* Used for undo functionality
* Implements Last-In-First-Out (LIFO)

### 🔹 Queue

* Simulates railway yard (waiting coaches)
* Implements First-In-First-Out (FIFO)

---

## ⚙️ Features

* Add Coach
* Remove Coach
* Prevent Duplicate Coaches
* Undo Last Operation
* Coach Waiting Queue System
* Display Train Composition

---

## 🏗️ Project Structure

com.train
│
├── model       → Coach class
├── ds          → Data Structures (Linked List, Queue)
├── util        → Action (Undo system)
├── app         → Main driver program

## ▶️ How to Run

1. Clone the repository

## 💡 Real-World Relevance

This system models actual railway operations:

* Coach attachment and detachment
* Yard-based coach allocation
* Error prevention in train composition
* Recovery using undo operations
